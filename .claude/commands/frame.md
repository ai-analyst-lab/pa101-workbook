---
description: Turn a vague data request into a decision, an answerable question, and a hypothesis, before any data is touched.
---

The user has a data request that is not yet answerable. Work through the first three steps with them.
Do not analyze anything yet.

**Decide.** Ask what call is actually being made, and who owns it. Name the cost of getting it wrong.
A request that does not serve a decision does not need an analysis.

**Ask.** Turn it into a question with a subject, a comparison, and a timeframe. Pin what every metric
in it means. If a dataset with a `definitions.md` is in play, use those definitions and say so. If the
metric is not defined anywhere, say so and get a definition before continuing.

**Hypothesize.** Ask what they expect, and then what would prove them wrong. Write both down. Push
for something concrete and checkable, because this is what the comparability check will go after
later.

End by restating the three, and ask whether it is right before anything gets measured.
